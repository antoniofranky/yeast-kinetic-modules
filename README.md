## Reproducing the analysis

### Prerequisites

- **Julia** ≥ 1.10 with [COCOA.jl](https://github.com/antoniofranky/COCOA.jl) and dependencies
- **R** ≥ 4.3 with packages: `tidyverse`, `ape`, `phytools`, `ggtree`, `caper`, `patchwork`, `here`, `readxl`, `ggrepel`, `xml2`
- **Python** ≥ 3.10 with: `pandas`, `numpy`, `matplotlib`, `scipy`, `statsmodels`, `openpyxl`

### Step 0 — Download large data from Zenodo

`data/prpd_models/no_split.tar.gz` (preprocessed SBML models without elementary-step splitting) ships directly in this repo — extract it to `data/prpd_models/no_split/`.

Download the rest from **[Zenodo DOI to be added]** and extract as described:

| Archive | Contents | Extract to |
|---|---|---|
| `prpd_models_random_0.tar.gz` | 332 preprocessed SBML models (main analysis) | `data/prpd_models/random_0/` |
| `prpd_models_variants.tar.gz` | Models for 10 seeds × 5 binding fractions | `data/prpd_models/variants/` |
| `jld2_random_0.tar.gz` | Kinetic results (JLD2, main analysis) | `data/jld2/random_0/` |
| `jld2_seeds_42-51.tar.gz` | Kinetic results for enzyme mechanism variants | `data/jld2/seeds/` |

### Step 1 — Model preparation (HPC, optional — results already in Zenodo)

Run on a SLURM cluster with COCOA.jl installed. Set environment variables:

```bash
export YEAST_GEMS_DIR=/path/to/Yeast-Species-GEMs
export PRPD_OUTPUT_BASE=/path/to/output
bash scripts/hpc/submit_pipeline.sh
```

The Yeast-Species-GEMs collection is available from [Shen et al. 2018](https://doi.org/10.1016/j.cell.2018.10.023).

### Step 2 — Kinetic analysis (HPC, optional — results already in Zenodo)

```bash
bash scripts/hpc/submit_kinetic_analysis.sh
```

### Step 3 — Extract summary tables from JLD2 results

```bash
# Set JLD2 result directories (or accept defaults after extraction to data/jld2/)
julia --threads=auto scripts/processing/01_kinetic_summary.jl
julia --threads=auto scripts/processing/02_giant_module_reactions.jl
julia --threads=auto scripts/processing/03_concordance_matrix.jl
```

### Step 4 — Reproduce figures

Run each script from the **repository root** (the `here` package uses the `.git` directory as the root anchor):

```r
Rscript scripts/figures/fig01_phylogeny_concordance.R   # Figure 1
Rscript scripts/figures/fig02_phenotype_correlations.R  # Figure 2 + PGLS data for Table S4
Rscript scripts/figures/fig03_pathway_enrichment.R      # Figure 3
```

Outputs are written to `figures/`.

### Step 5 — Build supplementary tables

```r
Rscript scripts/tables/tableS2_acr_metabolites.R        # Table S2 (requires Saccharomyces_cerevisiae.xml)
Rscript scripts/tables/tableS4_setup_data_prep.R
Rscript scripts/tables/tableS4_model_selection.R        # Table S4
python  scripts/tables/table1_pgls_results.py           # Table 1 (PGLS source data)
```

`fig02_phenotype_correlations.R` must be run before `table1_pgls_results.py` (it writes the PGLS CSV files to `results/pgls/`).
